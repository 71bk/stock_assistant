# 可觀測性與日誌

> 狀態：已落地基礎可觀測性 (2026-02-09)

## Log 策略

系統使用 **SLF4J + Logback**。

- 開發環境：Console log
- 生產環境：建議 rolling file + 集中式收集（ELK/Loki 皆可）
- 等級原則：
  - `ERROR`：未預期例外與 5xx
  - `WARN`：可恢復異常、第三方失敗、資料警示
  - `INFO`：業務主流程節點
  - `DEBUG`：僅開發排查使用

## TraceId / MDC

已實作 `TraceIdFilter`：

- 入口規則：
  - 若 request header 有 `X-Trace-Id` 則沿用
  - 否則自動產生新 `traceId`
- 寫入 `MDC`：key 為 `traceId`
- 回應 header：回傳 `X-Trace-Id`
- 請求完成後清理 `MDC`

## Metrics

基礎 metrics 已啟用：

- Actuator expose：
  - `/actuator/health`
  - `/actuator/metrics`
  - `/actuator/prometheus`
- 內建：JVM、HTTP、連線池等標準指標
- HTTP request histogram 已啟用，可計算 p50/p95/p99 延遲。
- Production Compose 內建 Prometheus：
  - 每 15 秒抓取 `backend:8080/api/actuator/prometheus`
  - 預設保留 30 天（`PROMETHEUS_RETENTION_TIME` 可調整）
  - 不對 host 發布 port
  - Nginx 明確封鎖公開的 `/api/actuator/prometheus`
- 股票代理自定義指標：
  - `stock.external.calls`
  - `stock.external.latency`
  - `stock.external.ratelimit.blocked`
  - `stock.cache.requests`（`hit/miss`）
  - `stock.singleflight.joined`

## Admin Analytics

- Admin 頁面可查：
  - 註冊人數、期間新增、DAU/WAU/MAU
  - SPA page views、sessions、熱門路由
  - API requests、4xx、5xx、error rate、p95 latency、熱門 endpoint
- 產品事件存於 PostgreSQL `app.analytics_events`，預設保留 90 天。
- 活躍使用者依 `PAGE_VIEW` 的 distinct `user_id` 計算，並排除 `ADMIN`。
- Prometheus label 不加入 user ID、Email、原始 URL 或其他高基數／PII 欄位。
- Prometheus 無法查詢時，使用者與頁面統計仍可正常顯示。

### AI / OCR Phase 0 Metrics

本階段只保存 Prometheus 原始用量，不建立成本資料表，也不估算美元：

- `ai_tokens_total{provider,model,operation,type}`
  - `type=input/output/cached`
- `ai_calls_total{provider,model,operation,success}`
- `ai_call_duration_seconds{provider,model,operation}`
- `ocr_fallback_total{path}`
- `ocr_retry_total{reason}`
- `embedding_retry_total{provider,model,scope,reason}`
- `embedding_batch_fallback_total{provider,model}`

固定 operation：

- `chat`：Groq 對話
- `analysis`：Groq 分析報告
- `ocr_vision`：Gemini/OpenAI/Ollama 圖像 OCR
- `ocr_parse`：OCR 文字轉交易 JSON
- `embedding`：RAG 文件與查詢向量化

限制與口徑：

- 不把 `user_id`、job ID、request ID 或錯誤訊息放入 label。
- Gemini `generateContent` 使用原生 `usageMetadata`。
- Groq streaming 使用最後的 usage chunk。
- Gemini embedding SDK 未提供可靠 token usage 時，只記 calls/duration/retry，不推估 token。
- Dashboard 顯示的是原始用量，不是供應商最終帳單或美元成本。

## 外部 API 日誌欄位

行情供應商呼叫（Fugle / Alpaca）統一記錄：

- `vendor`
- `endpoint`
- `status`
- `latency_ms`
- `request_id`
- `trace_id`

## Error Tracking

- Runtime error tracking 已接 Sentry（`sentry.dsn`）
- Build 時 source bundle upload 僅在 `SENTRY_AUTH_TOKEN` 存在時啟用

## 告警建議

- P0：服務無法啟動、DB/Redis 連線失敗
- P1：第三方行情連續失敗、`stock.external.ratelimit.blocked` 異常升高
- P2：OCR/AI 任務失敗率上升、延遲明顯惡化
